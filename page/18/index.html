linux的目录结构

发表于 2019-03-28 | 分类于 系统 |

TL;DR

  • /usr 只专门存放各种程序和数据
  • /var中包括了一些数据文件，如系统日志等。/var的存放使得/usr被只读挂载成为可能。
  • /usr/local
    新装的系统中这个文件夹是空的，可以用于存放个人安装的软件。安装了本地软件的/usr/local里的目录结构与/usr相似

误会

说起来，我也以为/usr就是user的意思，然后就觉得/home/xx好奇怪。
好奇心的谷歌了下。usr是unix system resources。
所以，用处不是某个用户。
现代的 /usr 只专门存放各种程序和数据。/usr 是系统核心所在，包含了所有的共享文件。它是 unix 系统中最重要的目录之一，涵盖了二进制文件，各种文档，各种头文件，还有各种库文件；还有诸多程序，例如 ftp，telnet 等等。

根文件系统

/bin
其中bin是binary的缩写，表示二进制文件，通常为可执行文件。一些常用的系统命令，如cp、ls等保存在该目录中。这一目录中存放了供所有用户使用的完成基本维护任务的命令。
/boot
这里存放的是启动Linux时使用的一些核心文件。如操作系统内核、引导程序Grub等。
/dev
在此目录中包含所有的系统设备文件。从此目录可以访问各种系统设备。如CD-ROM，磁盘驱动器，调制解调器和内存等。在该目录中还包含有各种实用功能，如用于创建设备文件的MAKEDEV。
/etc
该目录中包含系统和应用软件的配置文件。
/etc/passwd
该目录中包含了系统中的用户描述信息，每行记录一个用户的信息。
/home
存储普通用户的个人文件。每个用户的主目录均在/home下以自己的用户名命名。
/lib
这个目录里存放着系统最基本的共享链接库和内核模块。共享链接库在功能上类似于Windows里的.dll文件。
/lib64
64位系统有这个文件夹，64位程序的库。
/lost+found
这并不是Linux目录结构的组成部分，而是ext3文件系统用于保存丢失文件的地方。不恰当的关机操作和磁盘错误均会导致文件丢失，这意味着这些被标注为“在使用”，但却并未列于磁盘上的数据结构上。正常情况下，引导进程会运行fsck程序，该程序能发现这些文件。除了“/”分区上的这个目录外，在每个分区上均有一个lost+found目录。
/media
可移动设备的挂载点，当前的操作系统通常会把U盘等设备自动挂载到该文件夹下。
/mnt
临时用于挂载文件系统的地方。一般情况下这个目录是空的，而在我们将要挂载分区时在这个目录下建立目录，再将我们将要访问的设备挂载在这个目录上，这样我们就可访问文件了。（注意在GNOME中，只有挂载到/media的文件夹才会显示在“计算机”中，挂载到/mnt不会做为特殊设备显示，详见自动挂载分区）
/opt
多数第三方软件默认安装到此位置，如Adobe Reader、google-earth等。并不是每个系统都会创建这个目录。
/proc
它是存在于内存中的虚拟文件系统。里面保存了内核和进程的状态信息。多为文本文件，可以直接查看。如/proc/cpuinfo保存了有关CPU的信息。
/root
这是根用户的主目录。与保留给个人用户的/home下的目录很相似，该目录中还包含仅与根用户有关的条目。
/sbin
供超级用户使用的可执行文件，里面多是系统管理命令，如fsck, reboot, shutdown, ifconfig等。
/tmp
该目录用以保存临时文件。该目录具有Sticky特殊权限，所有用户都可以在这个目录中创建、编辑文件。但只有文件拥有者才能删除文件。为了加快临时文件的访问速度，有的实现把/tmp放在内存中。
/usr
静态的用户级应用程序等，见下。
/var
动态的程序数据等，见下文。
/usr目录结构
/usr通常是一个庞大的文件夹，其下的目录结构与根目录相似，但根目录中的文件多是系统级的文件，而/usr中是用户级的文件，一般与具体的系统无关。

usr

应注意，程序的配置文件、动态的数据文件等都不会存放到/usr，所以除了安装、卸载软件外，一般无需修改/usr中的内容。说在系统正常运行时，/usr甚至可以被只读挂载。由于这一特性，/usr常被划分在单独的分区，甚至有时多台计算机可以共享一个/usr。

/usr/bin
多数日常应用程序存放的位置。如果/usr被放在单独的分区中，Linux的单用户模式不能访问/usr/bin，所以对系统至关重要的程序不应放在此文件夹中。
/usr/include
存放C/C++头文件的目录
/usr/lib
系统的库文件
/usr/local
新装的系统中这个文件夹是空的，可以用于存放个人安装的软件。安装了本地软件的/usr/local里的目录结构与/usr相似
/usr/sbin
在单用户模式中不用的系统管理程序，如apache2等。
/usr/share
与架构无关的数据。多数软件安装在此。
/usr/X11R6
该目录用于保存运行X-Window所需的所有文件。该目录中还包含用于运行GUI要的配置文件和二进制文件。
/usr/src
源代码

var

/var中包括了一些数据文件，如系统日志等。/var的存放使得/usr被只读挂载成为可能。

/var/cache
应用程序的缓存文件
/var/lib
应用程序的信息、数据。如数据库的数据等都存放在此文件夹。
/var/local
/usr/local中程序的信息、数据
/var/lock
锁文件
/var/log
日志文件
/var/opt
/opt中程序的信息、数据
/var/run
正在执行着的程序的信息，如PID文件应存放于此
/var/spool
存放程序的假脱机数据（即spool data）
/var/tmp
临时文件

7.8.9月辅食的制作

发表于 2018-10-12 | 分类于 baby |

7月辅食制作公式

辅食泥的做法

辅食泥制作步骤: 

原料烹熟 =>    
    1. 勺子背压泥  
    1. 研磨碗捣鼓成泥  
    1. 压蒜器挤成泥状,特别适合叶类肉类

辅食泥做好后可以放冰格存起来,要的时候拿出来加热即可.
下面的面和粥就是,主食煮好之后,从冰箱拿出泥搅拌即可,很方便.

颗粒面的做法

颗粒面的制作步骤: 

面煮熟 => 加入配菜搅拌

原料: 面 + 液体 + 配料 

1. 颗粒面(GB10769 含铁)  
1. 液体(水,牛奶,高汤)
1. 配料(菜 蛋 肉 鱼 虾 坚果粉末))

比如: 南瓜青菜虾泥颗粒面的制作

1. 放颗粒面
2. 用水煮熟
3. 加入南瓜青菜虾泥搅拌

比如: 红薯油菜三文鱼鸡汤面

1. 放颗粒面
2. 放鸡汤煮熟
3. 加入红薯油菜三文鱼搅拌

十倍粥的做法

十倍粥: 10g粥 + 100g水 (水是粥的十倍)

制作步骤: 

粥煮熟 => 加入配菜

原料: 主食 + 液体 + 配料 

1. 粮食主食: 大米,小米,玉米,高粱米,红豆,绿豆,红薯
2. 液体: 水 奶 高汤
3. 配料: 菜 肉 蛋 水果 果干

比如: 西蓝花三文鱼大小米粥 

1. 大米小米加水煮熟
2. 然后加入西蓝花三文鱼煮一煮 也可以搅拌

比如: 苹果南瓜蛋黄米粥(生病的时候宝宝也会喜欢吃 甜甜的细腻的) 

1. 大米加水和一点奶粉煮熟
2. 然后苹果南瓜蛋黄煮一煮 或者搅拌

常用的配料: 土豆胡萝卜泥 青菜泥 火龙果 三文鱼 紫甘蓝泥 土豆鸡肉泥 肝粉

6-7月辅食和奶的安排

什么时候吃辅食

和奶一起喝
也可以和家人一起进餐,这样做饭时间短.

比如:
7.30喝奶
11.30辅食+喝奶
3.30辅食+喝奶
6喝奶
8喝奶

先吃辅食还是先喝奶

原则: 
1. 6-7月的辅食原则是不影响喝奶量,只是简单的尝味道以及补充部分营养,所以这阶段的辅食是很少量的
2. 辅食之后立即喝奶,因为如果分开,进食就会变成多次,会让宝宝的饥饿感不明显,这样降低宝宝对食物的兴趣,会觉得吃不吃无所谓,长期以往是不利于宝宝成长的

遇到问题的话参照的原则:
1. 如果吃辅食的时候情绪很差不愿意吃,可以先吃点点奶缓解宝宝的情绪(这里的量不多 仅仅为了缓解情绪),然后吃辅食,然后在吃奶吃到饱
2. 如果只想吃辅食不喝奶的话,那就调换顺序.
大原则就是: 6-7月龄的必须保证奶量.

辅食和奶需要多少

辅食一天1-2次即可.


进餐安排图片在微信

喝奶变少

6-7月龄如果因为辅食量影响奶量,就会影响宝宝的生长发育,所以奶量必须保证.
喝奶变少的话 就必须减少辅食量

8月龄辅食制作公式

8月龄食物的制作 - 辅食碎末(促进宝宝咀嚼)

碎末制作方法: 

1. 厨房搅馅器搅碎 (生的肉 菜 胡萝卜等) 再蒸或者煮熟
2. 弄熟之后 勺子戳碎 (土豆 胡萝卜 红薯类)
3. 菜刀剁碎
4. 辅食剪剪碎(面条 肉)

细面

汤面的一般步骤: 

面煮熟 - 加入配菜 - 剪成颗粒状

炒面的一般步骤: 

面煮熟 - 配菜炒熟 - 加入面条炒匀 - 剪成颗粒状

原料: 面(GB10769) + 液体(水 奶 汤) + 配料())

比如: 胡萝卜生菜虾肉面 

面(细面) + 液体(虾汤) + 配料(胡萝卜 生菜 虾肉))

比如: 番茄油麦菜肉末面 

面(细面) + 液体(水) + 配料(油麦菜 番茄 肉末)

面疙瘩

面疙瘩的三种制作方法: 

1. 漏勺法: 搅好面糊(放进有孔的漏勺里,用东西压下,这样面糊就顺着孔进锅里) - 煮熟疙瘩 - 加入配菜
1. 挖块法: 搅好面糊(面糊就是面和水搅拌,稍微干燥些,用小勺子挖进锅里) - 煮熟疙瘩 - 加入配菜
1. 滴水法: 滴水(打开水龙头 滴水到面粉里 一边滴一边摇匀)摇匀成疙瘩 - 煮熟疙瘩 - 加入配菜

原料: 粉 - 调糊 - 配料

1. 粉(面粉 可以加点全麦粉 玉米面 米粉 坚果粉之类)
2. 调糊 (水 奶 高汤 鸡蛋液 辅食汁[汁可以上色]  )
3. 配料(一堆)

比如: 南瓜泥青菜面疙瘩

原料: 粉(面粉 玉米面) + 调糊(南瓜泥 水) + 配料(青菜)

比如: 香菇巴沙鱼西蓝花鸡蛋面疙瘩

原料: 粉(面粉 婴儿米粉) + 调糊(紫甘蓝泥 水) + 配料(香菇巴沙鱼西蓝花鸡蛋)

肉丸

肉丸做好之后放入冰箱冷冻起来,需要的话拿出来蒸熟/煮熟即可.

一般步骤: 

肉打成泥 - 加入配料 - 塑性(圆形 片状 肉肠状) - 蒸熟/煮熟

其中圆形: 手搓即可
片状: 放在方形的容器蒸熟 拿出来切片即可
肉肠状: 肉肠状模具 蒸熟

原料: 肉 + 调糊 + 配料

1. 肉像(猪肉 牛肉 鸡肉  鸡肉 虾肉等)打成泥
2. 调糊(淀粉[玉米淀粉 马铃薯淀粉] 蛋液 奶 水 ),肉的粘性差加入前面那些增强粘性,如果肉的水分多就不用加水了
3. 配料(一点点即可 青菜泥 南瓜泥 菜的碎末 坚果的粉末等)

比如: 小白菜猪肉丸子 

原料: 肉(猪肉) + 调糊(淀粉) + 配料(小白菜 虾皮粉)

比如: 胡萝卜牛肉丸 

原料: 肉(牛肉) + 调糊(蛋清) + 配料(胡萝卜 土豆 香菇)

饼

一般步骤: 调面糊 - 烙熟

松饼

一般步骤: 

打发蛋白 - 混合面糊 - 烙熟面糊

1. 蛋清用电动打蛋器打成糊糊状
2. 蛋黄和面在一起
3. 激活酵母(放入温水) - 混合面糊 - 等待发酵 (30min) - 烙熟

原料: 粉 - 调糊 - 配料

1. 粉(面粉 可以加点全麦粉 玉米面 米粉 坚果粉之类)
2. 调糊 (水 奶 高汤 鸡蛋液 辅食汁[汁可以上色]  )
3. 配料(一堆)

比如: 菠菜胡萝卜饼 

原料: 粉(面粉) + 调糊(水) + 配料(菠菜 胡萝卜 甜椒))

比如:香葱酸奶玉米鸡蛋饼 

原料: 粉(面粉 玉米面) + 调糊(鸡蛋 酸奶) + 配料(香葱))

馒头/发糕

馒头步骤: 

揉面团 - 发酵 - 再揉下排气 - 二次发酵 - 蒸熟

原料: 

粉(面粉 可以加点全麦粉 玉米面 米粉 坚果粉之类) - 调糊 (安琪干酵母 奶 水 ) - 配料(紫薯泥 鸡蛋 南瓜泥 青菜泥)

比如: 馒头

原料: 粉(面粉 奶粉) - 调糊 (安琪干酵母 水 ) - 配料(胡萝卜汁 菠菜汁))

比如: 发糕 

原料: 粉(面粉) - 调糊 (安琪干酵母 水 ) - 配料(紫薯泥 山药泥))
