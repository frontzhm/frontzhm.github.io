不同于“老式”的传入回调，在使用 Promise 时，会有以下约定：

  • 在本轮 事件循环 运行完成之前，回调函数是不会被调用的。
  • 即使异步操作已经完成（成功或失败），在这之后通过 then() 添加的回调函数也会被调用。
  • 通过多次调用 then() 可以添加多个回调函数，它们会按照插入顺序进行执行。
  • Promise 很棒的一点就是链式调用（chaining）。

链式调用

连续执行两个或者多个异步操作是一个常见的需求，在上一个操作执行成功之后，开始下一个的操作，并带着上一步操作所返回的结果。我们可以通过创造一个 Promise 链来实现这种需求。

我觉得要理解 Promise，需要拆分成两个角度，综合理解

  • 平时怎么使用 Promise 的
  • JS 引擎怎么执行 Promise 代码的

怎么使用 Promise

Promise 本质是一个对象，它代表了一个异步操作的最终完成或者失败，可以在它上面绑定回调函数，这样我们就不需要在一开始把回调函数作为参数传入这个函数了。

像以前，请求的话，可能封装一个 ajax，然后将请求成功或者失败的回调一起入参：

换成 promise 的话，就相当于把异步操作和回调拆开：

所以总结下 Promise 的用法：

  • new Promise(executor)建一个 promise 实例，executor 是一个函数
  • 将异步操作的相关逻辑写在 executor 函数里，成功的处理调用用resolve(xxValue)，失败的处理用调用reject(xxReason)
  • 将异步操作完成之后的回调，另外传入then(value=>{xxx},reason=>{xx})

上面是用法，那，promise 接受了参数之后都做了什么呢，从执行的角度再看看

JS 引擎怎么执行 Promise 代码的

执行 executor 函数

  • 每个 promise 的初始状态是pending，初始值是undefined，初始错误理由也是undefined，初始成功函数数组是[]，初始失败函数数组是[]
  • 当执行到new Promise(executor)的时候，executor函数会立即执行
  • 在executor函数内，执行到resolve(xx)的时候，此时注意，不会立即执行resolve，而是会开启一个微任务队列，将resolve(value)丢过去
  • 同理，执行到reject的时候，也一样，开启一个微任务队列，将reject(reason)丢过去
  • 紧接着执行executor函数内其他的同步代码
  • 当executor函数执行完了，此时仅接着执行后面的then函数

执行 then 函数

注意，then函数可能有两个参数如.then(value=>{...},reason=>{...})，then 执行，不代表这两个函数执行。

then 执行，因为时机和返回值不一样，执行情况就会不一样，请仔细阅读下面的前置条件。

then执行的时候，先说此时，promise的状态是pending且then没有返回值的情况：

  • then(fn1,fn2)执行，fn1这个函数被收集进这个promise实例的成功函数数组，fn2被放进失败函数数组
  • 如果这个then后面还有then的话，重复上面的步骤
  • 所有的then都执行完之后，会接着执行页面剩余的同步代码
  • 页面的同步代码都执行完之后，开始执行宏任务队列里异步操作，异步操作成功之后调用resolve(value)，promise 的状态变成fulfilled，promise实例的value属性就变成此处的参数value，然后将 value 值传给，成功函数数组的第一个函数，函数执行，如果函数内没有异步操作，则 promise 值会变成函数返回值
  • 同理，若执行微任务队列里的reject(reason)，执行之后，·promise实例的reason属性就变成此处的参数reason，然后遍历执行失败函数数组
  • 但注意，状态只要不是pending，就会无视遇到的resolve或者reject函数
  • then执行，如果promise实例的状态是fulfilled的话，执行前面的函数，且 promise 的值变成此函数的返回值，同理rejected的话，执行后面的函数，且 promise 的 reason 变成此函数的返回值

最核心的地方：resolve 函数不会立即执行，而是放进微任务队列里，等着页面的同步代码执行完才去执行。

promise 是一个对象，它代表了一个异步操作的最终完成或者失败。

promise 对象用 state 属性来表示成功或者失败，值是fulfilled的时候表示异步操作的成功，是rejected表示异步操作的失败。
