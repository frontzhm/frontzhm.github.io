editorconfig让代码美化更加省心

发表于 2017-11-20 | 分类于 编辑器 |

简介

官网网址
EditorConfig让编辑器读取配置文件并依此格式化代码。
比如保存的时候,编辑器会自动在文件末尾添加新的一行.
有些编辑器会默认支持, 有些需要安装响应的插件, 官网有说明.

就近原则

当打开一个文件时，EditorConfig插件会在打开文件的目录和其每一级父目录查找.editorconfig文件，直到有一个配置文件root=true。
EditorConfig配置文件从上往下读取，并且路径最近的文件最后被读取。匹配的配置属性按照属性应用在代码上，所以最接近代码文件的属性优先级最高。一般放在项目根目录。

配置文件的名字是.editorconfig

支持的属性

  • indent_style：tab为hard-tabs，space为soft-tabs。
  • indent_size：设置整数表示规定每级缩进的列数和soft-tabs的宽度（译注：空格数）。如果设定为tab，则会使用tab_width的值（如果已指定）。
  • tab_width：设置整数用于指定替代tab的列数。默认值就是indent_size的值，一般无需指定。
  • end_of_line：定义换行符，支持lf、cr和crlf。
  • charset：编码格式，支持latin1、utf-8、utf-8-bom、utf-16be和utf-16le，不建议使用uft-8-bom。
  • trim_trailing_whitespace：设为true表示会除去换行行首的任意空白字符，false反之。
  • insert_final_newline：设为true表明使文件以一个空白行结尾，false反之。
  • root：表明是最顶层的配置文件，发现设为true时，才会停止查找.editorconfig文件。

常用的配置

文件匹配规则

通配符 说明
* 匹配除/之外的任意字符串
** 匹配任意字符串
? 匹配任意单个字符
[name] 匹配name字符
[!name] 匹配非name字符
{s1,s3,s3} 匹配任意给定的字符串（0.11.0起支持）

理解async

发表于 2017-10-28 | 分类于 js |

TL;DR

  • promise 对象最重要的两个属性,状态和值
  • async 函数返回promise对象, 函数不出错的话默认就是resolved状态, return的值就是promise的值
  • await 必须在async函数中, 后面是promise对象, 返回值是promise对象的值
  • await 必须等到promise的状态改变, 才会执行下一条语句

先回忆下promise

  • promise对象有两个属性, 状态和值,初始状态和值都是undefined
  • 一旦建立, 立即执行
  • 执行遇到resolve,状态就是resolved,值是resolve的第一个参数,reject同理

上例中,timeout(500,'hello')就表示此promise对象的状态是resolved, 值是hello

async函数

  • async函数返回一个 Promise 对象。
  • 返回的promise对象的状态默认是resolved, 值是函数的返回值
  • 特殊: async函数只有内部抛出错误的时候, 这个Promise对象的状态是rejected,值是错误对象

上例中, t就是promise对象, 状态是resolved, 值是 我就是值

await语句

  • await 命令只能用在 async 函数内部，如果用在普通函数，就会报错
  • 正常情况下，await命令后面是一个 Promise 对象。如果不是，会被转成一个立即resolve的 Promise 对象。类似执行了Promise.resolve()
  • await的返回值是promise对象的值
  • 特殊情况下, 如果await后面的promise对象的状态是rejected, 那么会报错, 所以通常async函数执行放在try,catch里, 一旦报错就会走catch

那么x这个promise对象状态是resolved, 值也是asyncAndPromise

学习node

TL;DR

  • 模块暴露module.exports, 引入模块require()
  • 全局对象global, 常用的其属性console和process, 下一次循环用process.nextTick(),process.argv获得命令行参数, 判断是不是浏览器环境typeof window === 'undefined'
  • fs模块, 同时提供异步和同步方法
    • 读文件: fs.readFile('sample.txt','utf-8',function(err,data){ 一般err为空 }), 二进制文件不用编码
    • 写文件: fs.writeFile('sample.txt','hi lalala',function(err){}), 字符串转bufferBuffer.from('hihi','utf-8'), buffer转字符串 bufferData.toString(‘utf-8’)
    • 得到文件的详细信息：fs.stat('sample.txt',function(err,data){data.isFile(),data.isDirectory(),data.size,data.birthtime,data.mtime})
  • stream是一个对象，有自己的事件：data事件表示流的数据已经可以读取了，end事件表示这个流已经到末尾了，没有数据可以读取了，error事件表示出错了。
    • 读文件流：var stream = fs.createReadStream('sample.txt','utf-8');stream.on('data',function(chunk){..});stream.on('end',function(){..});stream.on('error',function(err){..});，data事件可能会有多次，每次传递的chunk是流的一部分数据
    • 写文件流：var stream1 = fs.createWriteStream('sample.txt','utf-8');stream1.write('lala',function(){写完之后的回调});stream1.write((new Buffer('buffer','utf-8')));stream1.end()
    • pipe：把一个文件流和另一个文件流串起来，这样源文件的所有数据就自动写入到目标文件里了stream.pipe(stream1)
  • url.parse(urlStr,true)解析url
  • path.resolve([...paths])方法会把一个路径或路径片段的序列解析为一个绝对路径。
  • path.join([...paths]) 方法使用平台特定的分隔符把全部给定的 path 片段连接到一起，并规范化生成的路径。

用mockjs模拟数据

发表于 2017-09-26 | 分类于 js |

TL;DR

  • 安装 npm install mockjs
  • 数据模板定义规范
    • 属性名和生成规则必须用引号包围, 用|隔开
    • 属性值类型:number,string,boolean,object,array,function,regexp
    • 生成规则主要是 'name|min-max': value和'name|count': value, 属性值类型决定其含义
    • number: min-max之间的随机数   无后者不过加了浮点类型
    • string: min-max之间的随机重复次数   重复次数count
    • boolean: value出现的概率是 min/(min+max)    false和true均为1/2
    • object: 随机抽取min-max个属性   抽取count个属性
    • array: min-max之间的随机重复次数    1的话表示随机抽取一个值,其他值表示重复count次
    • function: 就是执行函数,this是属性所在的对象
    • regexp: 用正则生成字符串
  • 数据模板定义规范:用 @ 来标识其后的字符串是占位符 且占位符优先使用数据模板 中的属性
  • Mock.mock( rurl?, rtype?, template|function( options ) ) 生成数据
  • Mock.random() 占位符 随机生成想要的数据,比如email

h5和app交互的一些坑

发表于 2017-09-14 | 分类于 js |

安卓不兼容js最新的方法

这个特别注意，一不小心就犯了这个错。果子一般都支持。所以如果果子可以，安卓不可以，优先考虑这个问题

window对象延迟的问题

安卓一般情况下，会注入一个对象，这个在页面一开始加载就有了，比如安卓在window上注册了某个方法，基本可以直接使用。
不过偶尔安卓工程师可能会手动改这个注入顺序，这个需要沟通。

但是苹果，肯定是延迟注入的，是等页面加载完之后的不定时注入对象，自测了下不太准，50ms左右吧。
比如苹果提供某个方法，不能一开始就直接使用，需要延迟使用，一般监测到这个方法存在了，才能使用。

参数不对的问题

有时候h5如果传给app的参数不对的话，会出现各种问题。所以一定再三确认，参数没问题。
说起来，安卓不接受对象形式，所以一般用JSON。stringify转换对象传过去，但是果子可以。

脑洞各种意外的坑

如果别的安卓机都没问题，部分安卓机有问题，请发挥脑洞。
好像遇到过，魅族部分机型对于长字符串发送不了请求。。。
