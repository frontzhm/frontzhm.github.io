react的小入门

发表于 2017-05-18 | 分类于 js |

TL;DR

  • ReactDOM.render(html,container)
  • JSX 的语法允许 HTML 与 JavaScript 的混写，常用于数组，对了数组要加key
  • JSX遇到 HTML 标签（以 < 开头），就用 HTML 规则解析；遇到代码块（以 { 开头），就用 JavaScript 规则解析
  • 定义组件类:React.createClass(options),options里有render、getInitialState、propTypes、自定义属性等等

  • this.props.children表示组件的所有子节点

  • propTypes验证组件实例的属性是否符合要求，getDefaultProps 方法可以用来设置组件属性的默认值

  • refs获取真实的DOM节点

  • this.state将组件看成是一个状态机，一开始有一个初始状态，然后用户互动，导致状态变化，从而触发重新渲染 UI，getInitialState 方法用于定义初始状态，也就是一个对象，这个对象可以通过 this.state 属性读取。当用户点击组件，导致状态变化，this.setState 方法就修改状态值，每次修改以后，自动调用 this.render 方法，再次渲染组件。

  • 表单的value值，想要互动的话，需要定义在this.state
  • 组件有自己的周期
  • 组件的数据来源，通常是通过 Ajax 请求从服务器获取，可以使用 componentDidMount 方法设置 Ajax 请求，等到请求成功，再用 this.setState 方法重新渲染 UI

vue的表单控件绑定

发表于 2017-05-10 | 分类于 js |

TL;DR

  • v-model在表单控件元素上创建双向数据绑定。v-model="message"双向的意思是，input这边的输入值变化会让message变化，而message变化会让input的值变化。
  • v-model在文本框内就是文本框的值
  • v-model在复选框里，默认是布尔值，当有多个复选框时，要设置为数组，选中某项则数组中就会出现此项的值
  • v-model在选择列表里，放在select标签上，等于选中的option的值
  • v-model一般是input事件，修饰符lazy，可以将其变成change事件
  • 修饰符三种：lazy、number、trim（基本都是文本框）

vue的事件处理器

发表于 2017-05-09 | 分类于 js |

TL;DR

  • 直接写js代码：v-on:click=”counter += 1”
  • 方法事件处理器：v-on:click=”greet”
  • 内联处理器，需要event的时候必须传$event：v-on:click=”greet(param,$event)”，需要注意的是内联处理器必须是appvue实例里声明的方法
  • 事件修饰符：v-on:click.stop.prevent.once.self.capture=”greet”
  • 按键修饰符：v-on:keyup.enter.left.right=”greet”
  • 事件处理器的好处：一目了然，不需要找选择器，不需要时会自动销毁

vue的列表渲染

发表于 2017-04-24 | 分类于 js |

我们用 v-for 指令根据一组数组的选项列表进行渲染。 v-for 指令需要以 item in items 形式的特殊语法， items 是源数据数组并且 item 是数组元素迭代的别名。

TL;DR

  • 用于数组,index是可选项：v-for="(item,index) in items"
  • 用于对象,key,index是可选项：v-for="(value,key,index) in obj",顺序和Object.keys一致
  • v-for可以用于整数迭代：v-for = "n in 5"
  • v-for和v-if同时使用的情况
  • 数组的变异和重塑
  • 数组的index设值和length截断数组失效，用splice解决
  • 重新显示排序后的数组，用computed和methods
