vue的cli使用

发表于 2019-05-27 | 分类于 js |

TL;DR

  • sudo npm install @vue/cli @vue/cli-service-global -g安装vue的命令
  • vue serve xx.vue或者vue build能对单个 *.vue 文件进行快速原型开发
  • 但是用vue create xx建成的项目，一般用npm run serve或者npm run build
  • 目录结构的配置
  • 在遇到问题去vue CLI官网查询即可

vue cli（另外一篇子菜单也用了这个,懒得改了）

准备工作，安装vue CLI

vue serve，会在当前目录自动推导入口文件——入口可以是 main.js、index.js、App.vue 或 app.vue 中的一个。你也可以显式地指定入口文件vue serve xx.vue。
vue build 将目标文件构建成一个生产环境的包并用来部署。会在根目录里生成一个dist的包，里面有index.html css/ js/。

vue创建目录

vuecli官网说明，其实没啥，就是一个命令快速生成目录结构。在命令过程中，可以自定义选择，默认的有babel和eslint。

  • 选择过程中，空格是决定选中与否，上下键切换选项，回车之后进入下一个选择
  • In dedicated config files表示是否将配置项单独生成文件，一般我习惯用这个
  • 在选择要不要保存你刚刚的这些设置，保存的话可以code ~/.vuerc看看，以及编辑

在这个项目中，@vue/cli-service 安装了一个名为 vue-cli-service 的命令。你可以在 npm scripts 中以 vue-cli-service、或者从终端中以./node_modules/.bin/vue-cli-service 访问这个命令。一般用npm run serve或者npx vue-cli-service serve

vue.config.js 里的 devServer 字段配置开发服务器
npx vue-cli-service help [command]查看某个命令怎么用
npx vue-cli-service build --modern生成现代浏览器支持的包

遇到编译问题

cache-loader 会默认为 Vue/Babel/TypeScript 编译开启。文件会缓存在 node_modules/.cache 中——如果你遇到了编译方面的问题，记得先删掉缓存目录之后再试试看。

vuecli的插件

Vue CLI 使用了一套基于插件的架构。如果你查阅一个新创建项目的 package.json，就会发现依赖都是以 @vue/cli-plugin- 开头的。插件可以修改内部的 webpack 配置，也可以向 vue-cli-service 注入命令。在项目创建的过程中列出的特性，绝大部分都是通过插件来实现的。

可以远程预设配置

你可以通过发布 git repo 将一个预设配置分享给其他开发者。这个 repo 应该包含以下文件：

  • preset.json: 包含预设配置数据的主要文件（必需）。
  • generator.js: 一个可以注入或是修改项目中文件的 Generator。
  • prompts.js 一个可以为 generator 收集选项的 prompts 文件。
