vue结合pwa的使用

发表于 2019-07-23 | 分类于 vue |

项目来一个

直接打包用anywhere 启服务 看Application

目录

vue cli pwa
src/registerServiceWorker.js 这里面看到注册的名字
/public/manifest.json 看到缓存的详情

如果自己写了/dev/service-worker.js，打包之后的service-worker.js就是自己写的，不然就是框架生成的

vue.config.js那边配置名字，框架默认的名字是service-worker.js

修改名字 新建vue.config.js

vue cli pwa github

可以看下dist目录，service-worker.js，引入workbox内库，precache的url

自己写了/dev/service-worker.js，在dist目录里就会有自己的，但是会自动导入workbox和precahce.

增加缓存策略
workbox专门做缓存的库。

骨架屏

vue-skeleton-webpack-plugin
作者开发

skeleton.vue skeleton.js

dist/index.html 其实就是在div#app里面放入骨架屏的html代码而已，这样vue渲染出来之后，自然就替换掉了。

多个页面的话，可以多个骨架屏skeleton1.vue，然后再skeleton.js引入
