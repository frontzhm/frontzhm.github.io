用mockjs模拟数据

发表于 2017-09-25 | 分类于 js |

TL;DR

  • 安装 npm install mockjs
  • 数据模板定义规范
    • 属性名和生成规则必须用引号包围, 用|隔开
    • 属性值类型:number,string,boolean,object,array,function,regexp
    • 生成规则主要是 'name|min-max': value和'name|count': value, 属性值类型决定其含义
    • number: min-max之间的随机数   无后者不过加了浮点类型
    • string: min-max之间的随机重复次数   重复次数count
    • boolean: value出现的概率是 min/(min+max)    false和true均为1/2
    • object: 随机抽取min-max个属性   抽取count个属性
    • array: min-max之间的随机重复次数    1的话表示随机抽取一个值,其他值表示重复count次
    • function: 就是执行函数,this是属性所在的对象
    • regexp: 用正则生成字符串
  • 数据模板定义规范:用 @ 来标识其后的字符串是占位符 且占位符优先使用数据模板 中的属性
  • Mock.mock( rurl?, rtype?, template|function( options ) ) 生成数据
  • Mock.random() 占位符 随机生成想要的数据,比如email

语法规范

两种:数据模板定义规范和数据占位符定义规范

数据模板定义规范

数据模板中的每个属性由 3 部分构成：属性名、生成规则、属性值, 生成规则是可选的 ,属性名和规则需用|分开：

生成规则

生成规则 有 7 种格式：

  • ‘name|min-max’: value
  • ‘name|count’: value
  • ‘name|min-max.dmin-dmax’: value
  • ‘name|min-max.dcount’: value
  • ‘name|count.dmin-dmax’: value
  • ‘name|count.dcount’: value
  • ‘name|+step’: value

但是生成规则的具体含义却是由属性值类型确定的, 换言之, 属性值的类型不一样, 同样的生成规则表示的含义不一样.

属性值

  • 属性值中可以含有 @占位符。
  • 属性值还指定了最终值的初始值和类型。
属性值 String类型
  1. ‘name|min-max’: stringVal

通过重复 stringVal 生成一个字符串，重复次数大于等于 min，小于等于 max。

  1. ‘name|count’: stringVal

通过重复 stringVal 生成一个字符串，重复次数等于 count。

属性值 Number类型
  1. ‘name|+1’: number

属性值自动加 1，初始值为 number。用于数组生成.

  1. ‘name|min-max’: number

生成一个大于等于 min、小于等于 max 的整数，属性值 number 只是用来确定类型。

  1. ‘name|min-max.dmin-dmax’: number

生成一个浮点数，整数部分大于等于 min、小于等于 max，小数部分保留 dmin 到 dmax 位。

示例demo:

属性值 Boolean类型
  1. ‘name|1’: boolean

随机生成一个布尔值，值为 true 的概率是 1/2，值为 false 的概率同样是 1/2。

  1. ‘name|min-max’: value

随机生成一个布尔值，值为 value 的概率是 min / (min + max)，值为 !value 的概率是 max / (min + max)。

示例demo:

属性值 Object类型
  1. ‘name|count’: object

从属性值 object 中随机选取 count 个属性。

  1. ‘name|min-max’: object

从属性值 object 中随机选取 min 到 max 个属性。

示例demo:

属性值 Array类型
  1. ‘name|1’: array

从属性值 array 中随机选取 1 个元素，作为最终值。

  1. ‘name|+1’: array

从属性值 array 中顺序选取 1 个元素，作为最终值。

  1. ‘name|min-max’: array

通过重复属性值 array 生成一个新数组，重复次数大于等于 min，小于等于 max。

  1. ‘name|count’: array

通过重复属性值 array 生成一个新数组，重复次数为 count。

属性值 Function类型
  1. ‘name’: function

执行函数 function，取其返回值作为最终的属性值，函数的上下文为属性 ‘name’ 所在的对象。

数据占位符定义规范

占位符:只是在属性值字符串中占个位置，并不出现在最终的属性值中。
占位符 的格式为：

  • @占位符
  • @占位符(参数 [, 参数])
    注意:
  • 用 @ 来标识其后的字符串是 占位符。
  • 占位符 会优先引用 数据模板 中的属性。
  • 占位符 引用的是 Mock.Random 中的方法。
  • 通过 Mock.Random.extend() 来扩展自定义占位符。
  • 占位符 支持 相对路径 和 绝对路径。

Mock.mock( rurl?, rtype?, template|function( options ) )

根据数据模板生成模拟数据。

Mock.mock( template )

根据数据模板生成模拟数据。
可以用html文件,但必须有域名.

Mock.mock( rurl, template )

记录数据模板。当拦截到匹配 rurl 的 Ajax 请求时，将根据数据模板 template 生成模拟数据，并作为响应数据返回。

Mock.mock( rurl, function( options ) )

记录用于生成响应数据的函数。当拦截到匹配 rurl 的 Ajax 请求时，函数 function(options) 将被执行，并把执行结果作为响应数据返回。
option分get和post请求方式,options是默认被传进来的:

  • get请求, {url:url?key1=value1&key2=calue2, type:"GET", body:null}
  • post请求,{url:url, type:'POST', body:'key1=value1&key2=calue2'}

Mock.mock( rurl, rtype, template )

记录数据模板。当拦截到匹配 rurl 和 rtype 的 Ajax 请求时，将根据数据模板 template 生成模拟数据，并作为响应数据返回。
rtype是指get和post,同时符合rurl 和 rtype 的 Ajax 请求才会被Mock.

Mock.mock( rurl, rtype, function( options ) )

记录用于生成响应数据的函数。当拦截到匹配 rurl 和 rtype 的 Ajax 请求时，函数 function(options) 将被执行，并把执行结果作为响应数据返回。

参数的含义

rurl

可选。

表示需要拦截的 URL，可以是 URL 字符串或 URL 正则。例如 /\/domain\/list\.json/、’/domian/list.json’。

rtype

可选。

表示需要拦截的 Ajax 请求类型。例如 GET、POST、PUT、DELETE 等。

template

可选。

表示数据模板，可以是对象或字符串。例如 { 'data|1-10':[{}] }、’@EMAIL’。

function(options)

可选。

表示用于生成响应数据的函数。

options

指向本次请求的 Ajax 选项集，含有 url、type 和 body 三个属性，参见 XMLHttpRequest 规范。

Mock.setup()

配置拦截 Ajax 请求时的行为。支持的配置项有：timeout。

timeout

可选。

指定被拦截的 Ajax 请求的响应时间，单位是毫秒。值可以是正整数，例如 400，表示 400 毫秒 后才会返回响应内容；也可以是横杠 ‘-‘ 风格的字符串，例如 ‘200-600’，表示响应时间介于 200 和 600 毫秒之间。默认值是’10-100’。

Mock.Random

Mock.Random 是一个工具类，用于生成各种随机数据。

Mock.Random 的方法在数据模板中称为『占位符』，书写格式为 @占位符(参数 [, 参数]) 。

Web url, domain, email, ip, tld
Address area, region
Helper capitalize, upper, lower, pick, shuffle
Miscellaneous guid, id
