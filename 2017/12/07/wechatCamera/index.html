微信的图像开发接口

发表于 2017-12-07 | 分类于 js |

TL;DR

  • 必须配置微信参数,且在jsApiList配置你要的功能
  • vue历史模式下微信配置的坑
  • 微信拍照的官方文档
  • 出错优先查看官方文档,附录5-常见错误及解决方法
  • wx.chooseImage 得到localid的数组
  • wx.getLocalImgData 将localid的图片转化得到其base64,但不同系统的手机转化成的base64略有不同,需要处理才能使用
  • wx.uploadImage 上传图片(locaiid)到微信服务器 返回其serverid,也就是media_id, 三天之后失效, 可以通过get请求http://file.api.weixin.qq.com/cgi-bin/media/get?access_token=ACCESS_TOKEN&media_id=MEDIA_ID 下载图片(地址栏输入也可以直接下载)
  • wx.downloadImage下载微信服务器的图片(serverid)到本地,返回其本地的localid
  • wx.previewImage 预览图片,需要http链接或者本地图片的localid(图片必须在本地)
  • 使用微信拍照和服务器两种交互的方法

手机里面的控制台
腾讯开发的类似谷歌控制台, 在手机里测试网页的时候很方便. 用法很简单, 引入文件然后初始化即可

基础工作, 配置微信参数

页面引入最新版微信js : <script src="http://res.wx.qq.com/open/js/jweixin-1.2.0.js"></script>
旧版的有兼容问题,这里不展开描述.

注意在vue的历史模式下,有个坑,ios请求服务端的接口必须是首次进入页面的url,如http://demo.com/index,安卓的依然是window.location.href.split('#')[0],微信的文档也有解释,还有这个vue的issue

调用chooseImage,返回的数据格式

其他也基本类似,不管成功或失败都会有errMsg,然后是其他数据

拍照或从手机相册中选图接口 wx.chooseImage

此接口返回图片的localId

获取本地图片接口 wx.getLocalImgData

此接口可以接受图片的localId返回图片的base64,必须一张张传

上传图片接口 wx.uploadImage

此接口可以接受图片的localId返回图片在微信服务器的serverId ,上传图片有效期3天,然后可以用微信多媒体接口下载图片到自己的服务器,serverId 即 media_id。但是好像有丢失的情况, 所以有时候不选用此方案
下载图片http请求方式: GET, 请求地址http://file.api.weixin.qq.com/cgi-bin/media/get?access_token=ACCESS_TOKEN&media_id=MEDIA_ID
也必须一张张传,这里有个坑,如果出现”uploadImage:fail”,出现这种localId.toString()

下载图片接口 wx.downloadImage

此接口可以接受图片的serverId返回图片下载后的本地的localId

预览图片 wx.previewImage

背景黑色,图片宽度或者高度100%, current为空的时候默认是urls的第一个
url的格式可以是http的绝对链接,也可以是本地图片的localid,ios还可以支持base64但是安卓不支持

和服务器的配合

通用的方法可以先定义

直接将图片传到微信服务器

先调用“拍照或从手机相册选择图片接口”—>选择成功图片后—>调用“上传图片接口”—>上传成功后(也就是图片上传到了微信服务器上)->拿到serverId,然后把serverId传给服务器 (服务器调用“下载图片接口”—>将图片下载到服务器存储。)

直接将图片base64传给服务器

先调用“拍照或从手机相册选择图片接口”—>选择成功图片后—>调用“获取本地图片接口”—>拿到图片的base64之后,将其传给服务器。
