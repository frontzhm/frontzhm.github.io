gulp常用的4个API

  • 捕获文件 gulp.src(globs[,options])
  • 输出文件 gulp.dest(path[,options])
  • 定义任务 gulp.task(name[,deps][,fn])
  • 监测文件 gulp.watch(globs[, options], tasks) 或 gulp.watch(globs[, opts, cb])

gulp.src(globs[,options])

用来获取文件stream(流)的.

文件流不是实体的文件,是一个虚拟文件对象流(Vinyl files)，这个虚拟文件对象中存储着原始文件的路径、文件名、内容等信息,之后用pipe()方法把流导入到你想要的地方,当然也可以把流写入到文件中.
globs是可以是单个glob,也可以是glob数组,glob是匹配文件的正则,glob文档

gulp.dest(path[,options])

把流中的内容写入到文件中.
一般先通过gulp.src()方法获取到我们想要处理的文件流，
然后把文件流通过pipe方法导入到gulp的插件中，最后把经过插件处理后的流再通过pipe方法导入到gulp.dest()中，也就是把流中的内容写入到文件中,把文件流写入文件后，文件流仍然可以继续通过pipe使用

生成的文件路径和path之间的关系

path生成的文件路径是我们传入的path参数后面再加上gulp.src()中有通配符开始出现的那部分路径.

举个栗子说明下:
已知: gulp.src(‘script//*.js’),gulp.dest(destPath),
那么:生成的路径 = destPath//*.js

没有通配符的话
已知: gulp.src(‘script/aMoudle/a.js’),gulp.dest(destPath),
那么:生成的路径 = destPath/a.js

理解base

想要改变默认生成的文件路径,也是可以滴,配置gulp.src()方法参数中的base属性。其实base是一直存在的,不设置的话其有默认值
script/**/*.js的base默认是script,输出的时候将base换成destPath
script/aMoudle/a.js的base默认是script/aMoudle,输出的时候也是将base换成destPath
如果设置base的话,输出的时候也是一样的将src中的base换成destPath

gulp.task(name[,deps][,fn])

定义任务,deps是当前定义的任务需要依赖的其他任务，为一个数组,当前定义的任务会在所有依赖的任务执行完毕后才开始执行。如果没有依赖，则可省略这个参数,fn就是这个任务做的事,是个函数

deps里面的任务,只有同步编程才会按顺序,异步的时候,一个没有执行完下一个会直接开始.

那怎么等待异步任务中的异步操作完成后再执行后续的任务？

有三种方法可以实现：
第一：在异步操作完成后执行一个回调函数来通知gulp这个异步任务已经完成,这个回调函数就是任务函数的第一个参数。

第二：定义任务时返回一个流对象。适用于任务就是操作gulp.src获取到的流的情况。

第三:返回一个promise对象，例如

gulp.watch(globs[, opts], tasks) 或 gulp.watch(globs[, opts, cb])

监测glob文件,当其发生变化的时候就返回一个change的事件发射器,且可用它来执行相应的任务
gulp.watch(globs[, opts], tasks)监测到任务发生变化的时候,完成tasks(数组)里面的任务
gulp.watch(globs[, opts, cb])监测到文件发生变化的时候,执行回调函数,且传递一个event对象,来描述发生的改变
event.type:字符串类型,发生改变的类型有:added,changed,deleted,renamed
event.path:让事件发生的文件路径

gulp插件

复制文件,合并任务,不需要插件.
gulp本身可以读取文件、写入文件以及监控文件,所以能复制文件和合并任务
但是很多功能自身不能实现,需要依靠插件

常用的插件:

  • 编译 Sass：gulp-sass
  • 编译 Less：gulp-less
  • 合并文件：gulp-concat
  • 压缩js文件：gulp-uglify
  • 重命名文件：gulp-rename
  • 优化图像大小：gulp-imagemin
  • 压缩css 文件：gulp-minify-css
  • 压缩html 文件：gulp-minify-html
  • 创建本地服务器/实时预览：gulp-connect

插件使用顺序:

  1. npm install xxx --save-dev 安装插件
  2. 在 gulpfile.js 顶部引入此插件
  3. 在创建任务的时候使用此插件
  4. 具体用法去node搜文档
  5. 每安装一个插件都会加在package.json里

特别的 : gulp-load-plugins,gulp-jshint, 具体的自行搜索文档
